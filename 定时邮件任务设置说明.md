# 定时邮件任务设置说明

本文档用于说明如何在 Windows 上设置每天早上 9 点自动生成行业信息日报，并发送到配置好的邮箱。

## 前置条件

请先确认项目可以手动发送邮件：

```powershell
python -m src.send_mail
```

如果只是测试邮件发送，不想重新采集数据，可以运行：

```powershell
python -m src.send_mail --skip-generate
```

邮箱配置文件为 `.env.local`，该文件不会提交到 Git。请确认里面已经配置好 SMTP、发件人和收件人信息。

## 项目路径

当前项目路径为：

```text
C:\Users\Lenovo\Desktop\行业信息获取-定时邮件-codex
```

定时任务将调用项目内的脚本：

```text
C:\Users\Lenovo\Desktop\行业信息获取-定时邮件-codex\run_daily_mail.ps1
```

## 创建 Windows 任务计划

1. 打开开始菜单，搜索并打开：

```text
任务计划程序
```

2. 点击右侧：

```text
创建任务
```

建议使用“创建任务”，不要使用“创建基本任务”，可配置项更完整。

## 常规设置

在“常规”页填写：

```text
名称：行业信息日报邮件
```

建议勾选：

```text
使用最高权限运行
```

运行方式可按需要选择：

```text
只在用户登录时运行
```

或：

```text
不管用户是否登录都要运行
```

如果选择“不管用户是否登录都要运行”，保存任务时可能需要输入 Windows 登录密码。

## 触发器设置

进入“触发器”页，点击“新建”。

填写：

```text
开始任务：按计划
设置：每天
开始时间：09:00:00
启用：勾选
```

点击“确定”保存触发器。

## 操作设置

进入“操作”页，点击“新建”。

填写：

```text
操作：启动程序
```

“程序或脚本”填写：

```powershell
powershell.exe
```

“添加参数”填写：

```powershell
-ExecutionPolicy Bypass -File "C:\Users\Lenovo\Desktop\行业信息获取-定时邮件-codex\run_daily_mail.ps1"
```

“起始于”填写：

```text
C:\Users\Lenovo\Desktop\行业信息获取-定时邮件-codex
```

点击“确定”保存操作。

## 条件设置

进入“条件”页。

如果是笔记本电脑，建议取消勾选：

```text
只有在计算机使用交流电源时才启动此任务
```

其他选项可保持默认。

## 设置页建议

进入“设置”页，建议勾选：

```text
允许按需运行任务
```

建议同时勾选失败重试：

```text
如果任务失败，按以下频率重新启动
```

推荐配置：

```text
重新启动间隔：10 分钟
尝试次数：3 次
```

## 保存并测试

点击“确定”保存任务。

在任务计划程序列表中找到：

```text
行业信息日报邮件
```

右键点击，选择：

```text
运行
```

运行后可检查是否收到邮件。

## 查看运行日志

定时脚本会将日志写入：

```text
dist\logs\
```

日志文件名类似：

```text
daily_mail_20260602_090000.log
```

如果没有收到邮件，优先查看最新日志文件。

## 常见问题

### 任务运行后没有邮件

可以先手动运行：

```powershell
python -m src.send_mail --skip-generate
```

如果手动也失败，通常是邮箱 SMTP 配置、授权码、收件人地址或网络问题。

### 任务运行时间很长

正常。正式脚本会先重新采集和生成 HTML，再发送邮件。采集过程中会访问官网、搜索页、微信公众号、PDF 等来源，耗时可能较长。

### 只想快速重发已有日报

可以手动运行：

```powershell
python -m src.send_mail --skip-generate
```

定时任务不建议使用 `--skip-generate`，否则每天发送的可能不是最新日报。

### 换电脑后需要重新设置吗

需要。Windows 任务计划是系统级配置，不会随 Git 代码自动迁移。

换电脑后需要：

1. 安装 Python 和依赖。
2. 拷贝或重新创建 `.env.local`。
3. 确认 `python -m src.send_mail` 可以手动发送。
4. 按本文档重新创建任务计划。

## 新电脑部署详细步骤

本节用于说明将项目迁移到一台新 Windows 电脑后，需要安装的软件、复制的文件和验证步骤。

### 1. 安装 Python

建议安装 Python 3.11 或更高版本。

下载地址：

```text
https://www.python.org/downloads/windows/
```

安装时务必勾选：

```text
Add python.exe to PATH
```

安装完成后，打开 PowerShell，检查版本：

```powershell
python --version
```

如果能看到类似下面的输出，说明 Python 已可用：

```text
Python 3.x.x
```

### 2. 安装 Git

如果需要从 Git 仓库拉取项目，需要安装 Git。

下载地址：

```text
https://git-scm.com/download/win
```

安装完成后，打开 PowerShell 检查：

```powershell
git --version
```

如果能看到版本号，说明 Git 已可用。

### 3. 获取项目代码

如果项目已经推送到远程 Git 仓库，在新电脑上执行：

```powershell
git clone 仓库地址
```

进入项目目录：

```powershell
cd 行业信息获取-定时邮件-codex
```

如果没有远程仓库，也可以直接复制整个项目文件夹到新电脑。

注意不要只复制 `src`，至少需要包含：

```text
config\
src\
tests\
需求\
requirements.txt
README.md
run_daily_mail.ps1
.env.example
```

### 4. 创建虚拟环境

建议在项目目录下创建虚拟环境：

```powershell
python -m venv .venv
```

激活虚拟环境：

```powershell
.\.venv\Scripts\Activate.ps1
```

如果 PowerShell 提示不允许执行脚本，可以先运行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

然后重新激活：

```powershell
.\.venv\Scripts\Activate.ps1
```

激活成功后，命令行前面通常会出现：

```text
(.venv)
```

### 5. 安装项目依赖

在项目根目录执行：

```powershell
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

如果安装速度较慢，可以使用国内镜像：

```powershell
python -m pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 6. 配置邮箱环境文件

`.env.local` 不会提交到 Git，所以新电脑上需要重新创建。

可以先复制模板：

```powershell
Copy-Item .env.example .env.local
```

然后用记事本打开：

```powershell
notepad .env.local
```

根据实际情况填写 SMTP 配置、发件人、收件人。

163 邮箱通常使用：

```text
SMTP_HOST=smtp.163.com
SMTP_PORT=465
SMTP_USE_SSL=true
```

密码应填写邮箱的 SMTP 授权码，不是邮箱登录密码。

注意：

```text
.env.local 包含邮箱授权码，不要提交到 Git，不要发送给无关人员。
```

### 7. 手动测试生成日报

先测试生成 HTML 和 JSON：

```powershell
python -m src.main
```

成功后会生成：

```text
dist\industry_mail_preview.html
dist\industry_items.json
```

### 8. 手动测试发送邮件

完整生成并发送：

```powershell
python -m src.send_mail
```

如果只想测试 SMTP 和邮件发送，不重新采集：

```powershell
python -m src.send_mail --skip-generate
```

确认收件邮箱可以收到邮件后，再配置 Windows 任务计划。

### 9. 测试定时脚本

项目里已经提供定时脚本：

```text
run_daily_mail.ps1
```

在 PowerShell 中测试：

```powershell
powershell.exe -ExecutionPolicy Bypass -File ".\run_daily_mail.ps1"
```

如果运行成功，说明任务计划也可以调用该脚本。

### 10. 配置 Windows 任务计划

按本文档前面的“创建 Windows 任务计划”章节设置。

新电脑上的路径可能不同，需要把路径替换成新电脑实际项目路径。

例如新电脑项目路径是：

```text
D:\Projects\行业信息获取-定时邮件-codex
```

则“添加参数”应改为：

```powershell
-ExecutionPolicy Bypass -File "D:\Projects\行业信息获取-定时邮件-codex\run_daily_mail.ps1"
```

“起始于”应改为：

```text
D:\Projects\行业信息获取-定时邮件-codex
```

### 11. 查看日志

运行日志位于：

```text
dist\logs\
```

如果任务计划显示已运行但没有收到邮件，先查看最新日志。
